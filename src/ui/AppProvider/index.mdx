# AppProvider: Application Root Context

The `AppProvider` component is the mandatory, **top-level wrapper** for the entire React application. Its primary function is to initialize the necessary global contexts and apply baseline CSS standards, ensuring all child components have access to the unified design system and required application state.

---

## I. Purpose and Function

The `AppProvider` centralizes essential application setup logic, which includes:

* **Context Provision:** It establishes and provides critical React contexts to the component tree.
* **CSS Baseline Initialization:** It injects global CSS variables and standardizes base element styles, ensuring consistency across browsers.

---

## II. Usage Protocol

### Basic Integration

The `AppProvider` must wrap the primary `App` component within your main application entry point (e.g., `index.tsx` or `main.tsx`).

```tsx
// Example: src/index.tsx

import React from 'react';
import ReactDOM from 'react-dom/client';
// Import the core Provider
import { AppProvider } from '@emingy/core/ui'; 
import App from './App';

const root = ReactDOM.createRoot(document.getElementById('root') as HTMLElement);

root.render(
  <React.StrictMode>
    {/* The application MUST be wrapped by AppProvider */}
    <AppProvider>
      <App />
    </AppProvider>
  </React.StrictMode>
);
